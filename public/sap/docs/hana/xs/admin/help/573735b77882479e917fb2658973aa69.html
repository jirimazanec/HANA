Configure Any SAML 2.0 Identity Provider for an Identity Zone

An SAML identity provider is used by the SAML service provider to authenticate users signing in by means of a single sign-on mechanism.

Context

SAP HANA supports the use of single sign-on (SSO) authentication based on Security Assertion Markup Language 2.0 (SAML). An SAML identity provider is used by an SAML service provider to authenticate users who sign in to an application by means of single sign-on. In the identity provider, you want to establish a trust relationship with the SAML service provider. Perform the following configuration steps:

Procedure

  1. Upload the metadata for web-based authentication.
  2. Provide the attribute to identify the users. The SAML assertion includes the attribute in the form of a name ID.
  3. Include SAML groups in the SAML assertion. Include the names of your user groups in the assertion attribute Groups (capitalized).
  4. Test the SAML trust. You should be able to authenticate using the SAML identity provider. Use the following URL to see the user and the attributes in the output.

    https://<uaa_subdomain_name>.authentication.eu10.hana.ondemand.com/config?action=who
