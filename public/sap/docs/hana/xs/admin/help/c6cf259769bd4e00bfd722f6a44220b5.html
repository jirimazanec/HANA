Restage or Scale an XS Advanced Application

Configure the resources available for an application or restage an application droplet in the XS advanced run time.

Prerequisites

Access to the Application Monitor tool requires the authorization scopes defined in the roles grouped together in one of the following role collections:

  • XS_CONTROLLER_ADMIN

    Full access: no access restrictions

  • XS_CONTROLLER_USER

    Authorized to perform all tasks within an assigned organization

  • XS_CONTROLLER_AUDITOR

    Authorized to only view the details in an organization or space

Tip Role collections can be assigned to an SAP HANA user in SAP HANA studio by means of user parameters, for example, XS_RC_XS_CONTROLLER_ADMIN or XS_RC_XS_CONTROLLER_USER, or XS_RC_XS_CONTROLLER_AUDITOR.

Context

You can use the XS advanced administration tools to configure the resources available for an application and restage the application droplet in the corresponding run-time environment. Scaling an application means configuring the resources available to an application so that it can cope with greater demand, for example, by defining the number of instances allowed, the amount of memory (MB) allocated, and the disk space (MB) that is available for use by the application. Restaging an application stops the selected running application, creates a new application droplet using the a suitable buildpack, and starts the new droplet without updating the application sources.

To scale or restage an XS advanced application, perform the following steps:

Procedure

  1. Start the SAP HANA cockpit.
  2. Locate the XS Advanced Administration tools catalog.
  3. Start the Administration and Monitoring tools.
  4. Start the Application Monitor tool.
  5. Display a list of the applications running in the XS advanced run-time environment.
  6. Locate the application you want to manage and choose the desired action.

    In the Actions column of the list of available applications, choose more options (the list icon) and select one of the following actions:

    • Scale

      Configure the resources available for an application, for example, memory and disk space.

    • Restage

      Run the application's buildpack without updating the sources and start the new application droplet.

  7. Refresh the content of the application list.
    Tip You can enable the Auto Refresh feature to ensure that the latest application status is displayed.
