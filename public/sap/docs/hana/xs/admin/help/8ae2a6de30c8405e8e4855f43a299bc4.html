Application Monitoring Details

The information available in the Applications tab in the Application Monitor tool for XS advanced administrators.

The Applications tab in the Application Monitor tool enables you to view details of the XS applications running in the XS advanced run time that you are monitoring. The following table indicates which information can be viewed.

  • Application Information Display Type

  • Application Details

  • Application Actions

Information Display Type

You can use the grouping bar to group the information displayed according to certain criteria, for example, MTA or Space, as explained in the following table:

Tip Use the Settings button to choose what information to display, how to display it, and sort the displayed information according to any of the displayed details, for example: ascending or descending, Memory (KB) usage or CPU Time.
Table 1: XSA Application Details Display Type

Group Type

Description

None

Display all information by application in alphabetic order, starting with the letter “A”

MTA

Display the application list according to the contents of the deployed MTAs (Multi-Target Applications), for example, alm-product-installer: ver 1.0.0. Stand-alone applications that do not belong to an MTA are displayed together in the section Standalone Applications

Organization

Display the application list according to the organization to which they belong, for example, myorg or SAP

Space

Display the application list according to the space in which they are deployed and running, for example, myorg/SAP or myorg/PROD

Application Type

Display the application list according to deployment buildpack type, for example, Java, JavaScript, or Node.js

Application Actions

You can use the tools available in the Actions column to manage and maintain an application, as explained in the following table:

Tip Use the List button to choose what information to display, how to display it, and sort the displayed information according to any of the displayed details, for example: ascending or descending, Memory (KB) usage or CPU Time.
Table 3: XSA Application Details Display Type

Group Type

Description

Start

Starts the selected application

Stop

Stops the selected application

Restart

Stops and restarts the selected application; this refreshes the application instance without running the application's buildpack or updating the application source

Rename

Change the name of the XS advanced application

Restage

Stops the selected application, creates a new application droplet using the corresponding buildpack, and starts the new droplet.

Tip This action requires user interaction to confirm the restage-operation request

Scale

Configure the application to cope with greater demand by defining the number of instances, the amount of memory (MB) allocated, and the disk space (MB) that is available for use by the application

Pin to Host

Pin an application to one or more execution-agent hosts

Set Pin Mode

Specify the pinning mode for the selected application; the following modes are allowed:

  • “Strict”

    If the application is pinned to a host that is not available, the application cannot start.

  • “Relaxed”

    If no pinned host is available the application can start on another host.

Delete

Delete the selected XS advanced application, along with any mapped routes.
