Space Configuration Details

Details of the user spaces in an organization and their respective configuration.

In an organization, you define spaces to provide shared resources that can be used to develop, deploy, and maintain applications. Access to the shared resources is controlled by roles, for example, “Space Manager”, “Space Developer” or “Space Auditor”. The role defines the scope of the permissions assigned to the named user in the organizational space. For example, a Space Developer can deploy and start an application.

The Spaces pane in the Organization and Space Management tool displays the following information:

  • Host Pinning

    Information about the hosts pinned to the currently selected space

  • Users

    Information about the users configured for the selected space and the level of access they are allowed

  • Applications

    Information about the XS advanced applications running in the selected space

  • Services

    Information about the service and service instances available for use by XS advanced applications running in the selected space

Host Pinning

The Host Pinning tab in a selected Space displays the following information:

Table 1: Space Users

Field Name

Description

Name

The names of hosts pinned to the currently selected space

Mode

The pinning mode for the named host; the following modes are allowed:

  • “Strict”

    If an application is pinned to a host that is not available, the application cannot start.

  • “Relaxed”

    If no pinned host is available the application can start on another host.

Note If you want to set or change the pinning mode for a specific host in the current space, choose Set Mode.

Unpin

Remove the selected host from the list of hosts pinned to the currently active space
Users

The Users tab in a selected Space displays the following information:

Table 2: Space Users

Field Name

Description

User

The names of the XS advanced users currently configured for the selected space

Manager

Indicates if the user has been assigned the permissions associated with a space manager. Space managers can manage users in the selected space, view details of applications running in the space (status, instances, service bindings, and resource usage)

Auditor

Indicates if the user has been assigned the permissions associated with a space auditor. Space auditors can view details of applications running in the space (status, instances, service bindings, and resource usage)

Developer

Indicates if the user has been assigned the permissions associated with space developer. Space developers can deploy, start, or stop an application, bind or unbind an application and service, view details of applications running in the space (status, instances, service bindings, and resource usage)
