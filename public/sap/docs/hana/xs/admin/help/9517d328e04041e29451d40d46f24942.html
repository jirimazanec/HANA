Pin a Host to a Space

Pin one or more XS advanced hosts to a specific space.

Prerequisites

To access the XS advanced administration Spaces Management tool, a user requires the authorization scopes defined in the roles grouped together in one of the following role collections:

  • XS_CONTROLLER_ADMIN

    Full access: no access restrictions

  • XS_CONTROLLER_USER

    Modify and read-only access within an assigned organization or space

  • XS_CONTROLLER_AUDITOR

    Read-only access within an assigned organization or space

Tip Role collections can be assigned to an SAP HANA user in SAP HANA studio by means of user parameters, for example, XS_RC_XS_CONTROLLER_ADMIN or XS_RC_XS_CONTROLLER_USER, or XS_RC_XS_CONTROLLER_AUDITOR.

Context

To pin an XS Advanced host to a space in XS advanced, perform the following steps:

Procedure

  1. Start the SAP HANA cockpit.
  2. Locate the XS Advanced Administration tools catalog.
  3. Start the Organization and Space Management tool.
  4. Select the organization containing the space to which you want to pin a host.
  5. Pin a selected host to the currently active space.

    In the Host Pinning tab, choose Add, and in the Pin to Hosts dialog select from the list of hosts displayed the host (or hosts) that you want to pin the current space, and choose OK. All the applications in the space will be deployed into one of the pinned hosts.

  6. Set the pinning mode.
    You can set the pinning mode to “strict” or “relaxed” using the Set Mode button. The description of the modes are as below:

    • “Strict”

      If the application is pinned to a host that is not available, the application cannot start.

    • “Relaxed”

      If no pinned host is available the application can start on another host.

  7. Save your changes.

    In the Organization pane, choose Save.
