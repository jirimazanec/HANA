Add or Delete a Trusted Certificate in XS Advanced

Manage trusted certificates in XS advanced.

Prerequisites

To access the XS advanced administration Trusted Certificates tool, a user requires the authorization scopes defined in the roles grouped together in the following role collection:

  • XS_CONTROLLER_ADMIN

    Full access: no access restrictions

Tip Role collections can be assigned to an SAP HANA user in SAP HANA studio by means of user parameters, for example, XS_RC_XS_CONTROLLER_ADMIN or XS_RC_XS_CONTROLLER_USER, or XS_RC_XS_CONTROLLER_AUDITOR.

Context

The XS administration tool Trusted Certificates enables you to display a list of all certificates currently used to establish trusted connections between XS advanced systems. You can also use the tool to add or delete certificates, as described below:

Procedure

  1. Start the SAP HANA cockpit.
  2. Locate the XS Advanced Administration tools catalog.
  3. Start the Administration and Monitoring tools.
  4. Start the Trusted Certificates tool.
  5. Display a list of the trusted certificates currently used in the XS advanced run-time environment.
  6. Add a trusted certificate to the list of certificates.

    In the Trusted Certificates tool, choose Add, provide the following information :

    • Alias

      A name for the new trusted certificate; the name you provide here is displayed in the Alias column of the Trusted Certificates list.

    • Choose Certificate

      Browse to the file containing the certificate you want to add.

    • Usage

      Select the type of certificate used, for example: “HTTP” or “JDBC”. To set up trust with the SAP HANA database,use “JDBC”. The JDBC certificate establishes a secure and encrypted communication channel between SAP HANA XS applications and the SAP HANA database.

  7. Refresh the content of the certificates list to check that the new certificate is added to the list.
  8. Delete a certificate.

    In the Trusted Certificates tool, choose Delete.

  9. Refresh the content of the certificates list to check that the selected certificate was successfully removed from the list.
