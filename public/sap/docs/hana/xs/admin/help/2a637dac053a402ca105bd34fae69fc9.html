Organization Configuration Details

Details of the organizations configured in an XS advanced run-time environment.

Organizations enable users to collaborate by sharing resources, services, and applications. Access to the shared resources, services, and applications is controlled by roles, for example, “Org Manager” or “Org Auditor”; the role defines the scope of the permissions assigned to the named user in the organization. For example, an “Org Manager” can create and manage users, create, modify, or delete user spaces in an organization, and add domains to the organization.

The Organization pane in the Oranization and Space Management tool includes the following tools and tabs:

  • Spaces

    Information about the user spaces configured in the selected organization

  • Users

    Information about the users configured for the selected organization and the level of access they are allowed.

Spaces

The Spaces tab in a selected Organization displays the following information:

Table 1: Spaces in an Organization

Field Name

Description

Space

The name of the space configured in the selected organization, for example, the (default) “SAP” space or “PROD”.

Tip

Choose Create Space to add a user space to the selected organization.

Click a space in the list of spaces of choose > to display more information about an existing user space in the selected organization.
Users

The Users tab in a selected Organization displays the following information:

Table 2: Users Scopes in an Organization

Field Name

Description

User

The names of the XS advanced users currently configured for the selected organization

Manager

Indicates if the user has the permissions assigned to a space manager. Space managers can manage users in the selected space, view details of applications running in the space (status, instances, service bindings, and resource usage)

Auditor

Indicates if the user has the permissions assigned to a space auditor. Space auditors can Vvew details of applications running in the space (status, instances, service bindings, and resource usage)
