Using hdbalm with SAP HANA Cloud Platform

If you want to use hdbalm for SAP HANA systems in the SAP HANA Cloud Platform you must use secure communication (Secure Sockets Layer (SSL) protocol and certificates as described in the Enable SSL for hdbalm topic). This is required since systems in the SAP HANA Cloud Platform communicate using HTTPS.

If used in the SAP HANA Cloud Platform, hdbalm commands require the following options:

Option

Description

Sample Value

-h

host of the SAP HANA instance that is assigned to the account in the SAP HANA Cloud Platform

demo123.hana.ondemand.com

-p

port of the SAP HANA instance that is assigned to the account in the SAP HANA Cloud Platform

443

This is the only port that allows HTTPS communication.

-s

indicates that HTTPS communication is used

no value

-c

valid certificate stored in X.509 with chain (PEM) format

-.hana.ondemand.crt

Example

Example of an installation of the SAP HANA product archive DEMOPRODARCH.zip on the SAP HANA Cloud Platform instance demo123.hana.ondemand.com:
Sample Code 
hdbalm -u SYSTEM -h demo123.hana.ondemand.com -p 443 -s -c -.hana.ondemand.crt install DEMOPRODARCH.zip
Exporting a Certificate

To issue an hdbalm command in the SAP HANA Cloud Platform, a certificate is required to address the SAP HANA instance that is assigned to the SAP HANA Cloud Platform account for which you want to use hdbalm command. You canc export this certificate using the Web browserÂ´s export function.

Example

The following is an example of how to export an X.509 with chain (PEM) certificate in Mozilla Firefox:

  1. Start SAP HANA Application Lifecycle Management on the SAP HANA instance from which you want to export the certificate.

    Example: https://demo123.hana.ondemand.com/sap/hana/XS/LM

  2. Click on the Lock icon in the browser bar:

  3. Choose More Information....

  4. On the Security ribbon, choose View Certificate.

  5. On the Details tab, choose Export....

  6. Browse for a location where you want to store the file, and save the certificate as file of type X.509 with chain (PEM).

    The certificate is saved as *.crt file.
