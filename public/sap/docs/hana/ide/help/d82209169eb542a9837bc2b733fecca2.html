Tutorial: Create a Table

In this tutorial, you use the SAP HANA Web-based Development Workbench Editor to create a table as a design-time file in the repository using the hdbtable syntax. In the catalog, you view the table definition and insert data into the table.

Prerequisites

You have the privileges granted by the sap.hana.ide.roles::EditorDeveloper and sap.hana.ide.roles::CatalogDeveloper roles; these roles are included in the parent role sap.hana.ide.roles::Developer.

Procedure

  1. Open the SAP HANA Web-based Development Workbench Editor tool.
    The Editor tool is available on the SAP HANA XS Web server at the following URL: http://<WebServerHost>:80<SAPHANAinstance>/sap/hana/ide/editor
  2. Create a new application.
    1. From the context menu of the Content folder, choose Create Application.
    2. Choose the Empty application (with XSAccess and XSApp) template.
    3. Enter a package name, for example, demo.tables, and choose Create.
    4. In the context menu of the demo.tables package, choose Activate All.
  3. Create a schema.
    1. Select the newly created demo.tables package and from the context menu choose  New  File .
    2. Enter the name of the schema in the File Name field, for example, myschema.hdbschema, and choose Create.
    3. Add the schema definition by choosing (Insert snippet) in the toolbar and change the schema name to MYSCHEMA, for example, schema_name = “MYSCHEMA”;.
    4. Save the file and choose (Assign execution authorization) in the toolbar.
  4. Create the new table definition.
    1. Select the demo.tables package and from the context menu choose  New  File .
    2. Enter the name of the table in the File Name field, for example, mytable.hdbtable, and choose Create.
    3. Choose (Insert snippet) in the toolbar to insert the following table definition code. Remember to change the schema name in the inserted code to MYSCHEMA. 
      table.schemaName = "MYSCHEMA"; 
table.tableType = COLUMNSTORE; 
table.columns = [ 
    {name = "KEY1"; sqlType = TINYINT; nullable = false;}, 
    {name = "FIELD1"; sqlType = DOUBLE; nullable = false;}, 
    {name = "FIELD2"; sqlType = DOUBLE; nullable = false;}, 
    {name = "FIELD3"; sqlType = NVARCHAR; length = 10; nullable = false;} 
    ]; 
    table.primaryKey.pkcolumns = ["KEY1"];
    4. Save the file.
  5. In the catalog, check the table definition.
    1. To open the catalog, choose   (Navigation Links)   Catalog .
    2. Select the mytable file under the   MYSCHEMA  Tables  node.
      The table definition is displayed, as shown in the example below:

  6. Add entries to the table.
    1. Choose Open Content.
      The data preview appears, displaying an empty table.
    2. Choose (Insert).
    3. In the new row, enter some test data and save your entries, as shown in the example below:

  7. Generate a SQL select statement on the table.
    In the context menu of the mytable file, choose Generate Select.
    The table rows you just created are displayed in the SQL result view.
