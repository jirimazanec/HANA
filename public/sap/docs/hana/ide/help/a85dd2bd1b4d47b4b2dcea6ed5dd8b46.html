Debug Privileges for Procedures

To debug procedures, you require the DEBUG privilege. To debug procedures in an external session, you also need the ATTACH DEBUGGER privilege.

Privilege Description Use Case
DEBUG

(object privilege)

 Authorizes debug functionality on a specific procedure or on the procedures of a specific schema. It allows the user to:
  • Set breakpoints
  • Display the procedure source in the procedure viewer
  • Inspect intermediate results
 Required to debug procedures that are owned by another user.

Procedures created in the repository are owned by _SYS_REPO and the user therefore needs to be granted the debug authorization.

Procedures created directly in the catalog using the SQL console are owned by the user creating them, so that user does not require an additional debug authorization to debug the procedure.

ATTACH DEBUGGER

(system privilege)

 Authorizes the debugging of a procedure called by a different user. In addition, the DEBUG privilege for the corresponding procedure is needed. Required to debug an external session belonging to a different user. The session owner needs to grant the ATTACH DEBUGGER authorization to the user who is debugging.
