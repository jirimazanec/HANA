Use Aliases for Table Names and Column Names

You can assign one or more alternate names (or aliases) to tables. For example, if you want to improve the readability of a table name or if you want to add the same table again to your data foundation node, then you can use aliases to avoid name conflicts.

Procedure

  1. Aliases for table names.
    If you want to use aliases for table names, then:
    1. Select a node with data source.
    2. In the Mapping tab, select a table.
    3. Unhide the Properties section.
    4. In the Alias text box, provide an alias name for the table.
  2. Aliases for column names.
    If you want to use alias names for column names, then:
    1. Select a node with data source.
    2. In the Mapping tab, select an output column.
    3. Unhide the Properties section.
    4. In the Name text box, provide an alias name for the column.
