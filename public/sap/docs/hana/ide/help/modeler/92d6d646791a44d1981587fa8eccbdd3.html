Root Node Visibility

Based on your business requirement, choose to add an additional root node to the hierarchy and place all other nodes as its descendants.

Root Node Visibility

Description

Add Root Node If Defined

This applicable only for parent-child hierarchies. Modeler adds a root node only if you have defined a root node value, while creating the parent child hierarchy.

Add Root node

The modeler adds an additional root node to the hierarchy and all other nodes are placed as descendants to this node. Select this value if your hierarchy does not have a root node, but needs one for reporting purposes. Modeler creates a root node with the technical name ALL.

Do Not Add Root Node

The modeler does not add an additional root node to the hierarchy.
