Debugging Procedures

The SAP HANA SQLScript debugger allows you to debug and analyze procedures.

In a debug session, your procedures are executed in serial mode, not in parallel (not optimized). The stored procedure call stack appears in the debug view, allowing you to view the nested calls and test the correctness of the procedure logic. Note that the debugger is not intended to be used for evaluating performance.
