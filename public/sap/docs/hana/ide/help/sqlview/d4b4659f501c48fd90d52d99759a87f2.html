ALTER WORKLOAD CLASS Statement (Workload Management)

Changes workload classes.

Syntax
ALTER WORKLOAD CLASS { 
   <workload_class_name> { <property_list> | ENABLE | DISABLE }
   | ALL { ENABLE | DISABLE } }
Syntax Elements
<workload_class_name>

Changes the specified workload class.

<workload_class_name> ::= <identifier>
<property_list>

Defines the properties of the specified workload class.

property_list ::= [SET | UNSET] {'<key>' = '<value>'}[,{'<key>' = '<value>'}] [...] 

<key> ::= 
 PRIORITY 
 | STATEMENT MEMORY LIMIT 
 | STATEMENT THREAD LIMIT

The properties are key-value pairs.

Despite the name, STATEMENT THREAD LIMIT should be thought of more as a statement concurrency limit than a thread limit; SAP HANA does not impose a thread limit on statements.

ENABLE or DISABLE

Enables or disables the specified workload class.

ALL
Changes all workload classes.
Description

The use of this statement requires the WORKLOAD_ADMIN privilege.

A workload class with only NULL values behaves like the default workload class.

Examples

Unset the value for the STATEMENT_MEMORY_LIMIT for an existing workload class MyWorkloadClass. The WORKLOAD_CLASSES view displays the value NULL for the column STATEMENT_MEMOY_LIMIT of this workload class. It also changes the PRIORITY to 5.

ALTER WORKLOAD CLASS "MyWorkloadClass"
   UNSET 'STATEMENT MEMORY LIMIT'
   SET 'PRIORITY' = '5';

Enable all workload classes and their related mappings.

ALTER WORKLOAD CLASS ALL ENABLE;
