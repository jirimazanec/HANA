BACKUP LIST DATA Statement (Backup and Recovery)

Searches for the most recent backup catalog.

Syntax
BACKUP LIST DATA [<for_database>] <until_definition> [<system_source_definition>] 
   [<log_data_path_definition>] [<limit_offset>]
 | BACKUP COMPLETE LIST DATA [<for_database>] [<system_source_definition>] [<log_data_path_definition>]
   [<limit_offset>]
Syntax Elements
<for_database>

Specifies the backup catalog for a tenant database. 

<for_database> ::= FOR <databasename>

This command is only available for SAP HANA tenant databases.

For example: BACKUP LIST DATA FOR DB0.

Requirements:

  • Executed on the system database only.

  • Privilege needed: DATABASE ADMIN.

  • The system database must be online, but the tenant databases must be offline.

<until_definition>
<until_definition> ::=
 UNTIL LOG POSITION <log_pos> AT VOLUME <volume_id>
 | UNTIL TIMESTAMP '<timestamp>'
<system_source_definition>
<system_source_definition> ::= USING SOURCE <system_id>
<log_data_path_definition>
<log_data_path_definition> ::= 
 [USING CATALOG BACKINT] [USING LOG PATH ( <path_list> )] [USING DATA PATH ( <path_list> )]

<path_list> ::= '<path>'| <path_list> , '<path>'
<limit_offset>
<limit_offset> ::= LIMIT <int_const> [OFFSET <int_const>]
Description

The most recent backup catalog available is searched at either the default log backup location or at the locations specified by <log_source_definition>. In addition, third-party backup tools are searched, unless you restrict the search to only the third-party backup tool (with USING CATALOG BACKINT).

The following options can be used with SAP HANA single database containers and SAP HANA tenant databases:

  • LIMIT <limit> restricts the number of elements returned.

    If LIMIT <limit> is not specified, then a default value of 10 is used.

  • OFFSET <offset> defines the number of data backups to skip at the beginning of the list.

    If OFFSET<offset> is not specified, then no data backups are skipped.
