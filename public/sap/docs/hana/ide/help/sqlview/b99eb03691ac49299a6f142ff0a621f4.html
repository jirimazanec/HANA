BACKUP CHECK ACCESS Statement (Backup and Recovery)

Checks whether backups are accessible.

Syntax
BACKUP CHECK ACCESS [<for_database>] BACKUP_ID <int_const>
   [<system_source_definition>] [<log_data_path_definition>]
Syntax Elements
<for_database>

Checks if a specified backup is accessible for the tenant database.

<for_database> ::= FOR <databasename>

This command is only available for SAP HANA tenant databases.

<int_const>
<system_source_definition>
<system_source_definition> ::= 
 USING SOURCE '<system_id>'
<log_data_path_definition>
<log_data_path_definition> ::=
 [USING CATALOG BACKINT]
 [USING LOG PATH ( <path_list> )]
 [USING DATA PATH ( <path_list> )]

<path_list> ::= 
 '<path>'| <path_list> , '<path>'
Description

This statement checks whether backups are accessible.

BACKUP CHECK ACCESS only checks an individual backup. The CHECK ACCESS option for RECOVER DATABASE checks whether all the backups required for a recovery are available.
