Full description

This view has a resettable counterpart. This means, you can also see the values since the last reset in the dependent view M_MUTEXES_RESET. To reset the values, use the SQL command: ALTER SYSTEM RESET MONITORING VIEW SYS.M_MUTEXES_RESET or the management console command: monitor reset -n M_MUTEXES_RESET.

This view contains information about single mutex (mutual exclusion) objects or groups of mutex objects. It does not contain information about all mutex objects. Information like LOCK_COUNT, WAIT_COUNT and WAIT_TIMES can be used to analyze performance bottlenecks. Using OWNER_ID you can find possible deadlocks.
