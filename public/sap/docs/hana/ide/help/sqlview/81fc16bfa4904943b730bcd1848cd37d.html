ALTER WORKLOAD MAPPING Statement (Workload Management)

Changes workload mappings.

Syntax
ALTER WORKLOAD MAPPING <mapping_name> 
   WORKLOAD CLASS <wc_name> <property_list>
Syntax Elements
<mapping_name>

Specifies the identifier for the workload mapping to be changed.

<mapping_name> ::= <identifier>
<wc_name>

Specifies the identifier for the workload class name.

<wc_name> ::= <identifier>
<property_list>

Defines the properties of a workload mapping.

<property_list> ::= [SET | UNSET] '<key>' = '<value>' [...]
 
<key> ::= 
 APPLICATION USER NAME 
 | CLIENT 
 | APPLICATION NAME 
 | USER NAME 

<value> ::= <string_literal>

The properties have the format of key-value pairs.

Description

The use of this statement requires the WORKLOAD_ADMIN privilege.

Examples

Change the definition of the workload mapping “MyWorkloadMapping” to the database user ‘ABCADM’ and unset the application 'APPLICATION NAME'.

ALTER WORKLOAD MAPPING "MyWorkloadMapping" WORKLOAD CLASS "MyWorkloadClass"
   SET 'USER NAME' = 'ABCADM' UNSET 'APPLICATION NAME';
