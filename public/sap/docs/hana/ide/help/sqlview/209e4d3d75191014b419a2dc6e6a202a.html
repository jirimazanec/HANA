AUDIT_POLICIES System View

Provides information about audit policies.

Structure

Column name

Data type

Description

AUDIT_POLICY_NAME

NVARCHAR(256)

Audit policy name

AUDIT_POLICY_OID

BIGINT

Object ID of the audit policy

EVENT_STATUS

VARCHAR(32)

Status of events to be audited: SUCCESSFUL EVENTS, UNSUCCESSFUL EVENTS, ALL EVENTS

EVENT_LEVEL

VARCHAR(16)

Level of events to be audited: EMERGENCY, CRITICAL, ALERT, WARNING, INFO

IS_AUDIT_POLICY_ACTIVE

VARCHAR(5)

Audit policy is active: TRUE, FALSE

IS_VALID

VARCHAR(32)

TRUE, FALSE ( ). An audit policy can be invalid if the referenced user or object does not exist or an invalid combination of action and object is specified (for example, EXECUTE on a table)

EVENT_ACTION

VARCHAR(32)

Action to be audited, for example: SELECT, GRANT PRIVILEGE, ...

USER_NAME

NVARCHAR(256)

User whose actions are to be audited

EXCEPT_USER_NAME

NVARCHAR(256)

User whose actions are not to be audited

OBJECT_TYPE

VARCHAR(32)

Type of object to be audited, INVALID in case of change of object type after creation of audit policy

OBJECT_SCHEMA

NVARCHAR(256)

Schema of object to be audited

OBJECT_NAME

NVARCHAR(256)

Name of object to be audited

TRAIL_TYPE

VARCHAR(6)

Name of the audit trail where the audit entry is written
