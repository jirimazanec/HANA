ALTER SYSTEM { ADD | REMOVE } STATEMENT HINT Statement (System Management)

Adds or removes statement hints from the system.

Syntax
ALTER SYSTEM ADD STATEMENT HINT ( <hint_list> ) FOR <target_query>
| ALTER SYSTEM REMOVE STATEMENT HINT [ ALL ] FOR <target_query>
Syntax Elements
<hint_list>

Specifies a set of hints will be applied to the target query.

<hint_list> ::= <hint> [,... ]
<target_query>

Specifies the target query.

ALL
Removes both user hints and system hints from the system that are associated with the specified statement. If ALL is not specified, then only the user-defined hint is removed.
Description

In a statement hint entry, there are two kinds of hints: system hints and user hints. User hints are the hints added by users with the ADD STATEMENT HINT defined above. System hints are the hints provided by the system by default when the system is upgraded.

In the case of conflicts, for example if user hints are already specified in the system for the SQL statement to which the system hints will be applied during the system upgrade, then the user hints stay in the system and the system hints are kept for later usage.

ADD STATEMENT HINT adds statement hints to the system with the specified SQL statement string as a key.

REMOVE STATEMENT HINT removes statement hints from the system that match with the specified SQL statement string. If there are system hints, and the ALL option is not specified, then instead of removing the statement hints from the system, REMOVE STATEMENT HINT replaces the current user hints with the system hints. If the ALL option is specified, the statement removes the user hints and the system hints together with the specified SQL statement string.

Both commands (ADD, REMOTE) also evict all cached plans for the target SQL statements from the SQL Plan Cache.
