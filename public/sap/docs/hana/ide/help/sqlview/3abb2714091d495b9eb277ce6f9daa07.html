CS_VIEW_PARAMETERS System View

Provides a list of parameters of the objects in SAP HANA Database. For now, only calculation views are considered. The parameters of a view are parsed from the definition of the underlying scenario.

Structure
Note A select * query on this monitoring view can be very expensive since the scenarios that have not been cached so far are fetched from the repository, are parsed and their parameters are extracted. However, the query time can be decreased remarkably if the SCHEMA_NAME and/or OBJECT_NAME is provided.

Column name

Data type

 Unit

Description

SCHEMA_NAME

 NVARCHAR(256)  

The schema in which the object is deployed

OBJECT_NAME

NVARCHAR(256)

  

The name of the object. For now, only the Calculation Views are listed. The Calculation Views are not shown if the underlying Calculation Scenario has no parameter

PARAMETER_NAME

NVARCHAR(256)

  

The name of the parameter. If a parameter is referenced by different views, then the parameters are listed for each view separately. Normally, the parameters in a calculation views begins and ends with “$$” character. For example: $$VAR$$

IS_MANDATORY

VARCHAR(5)

  

Indicates if a parameter is mandatory or not. Possible values are TRUE and FALSE. If TRUE, the value of the parameter should be provided either by setting the default value or through the SQL query. If FALSE, setting the value of the parameter is optional

DEFAULT_VALUE

NVARCHAR(256)

  

The default value of the parameter. This column is set to empty string when the parameter has no default value
