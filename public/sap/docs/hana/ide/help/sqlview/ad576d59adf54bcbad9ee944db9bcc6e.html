DROP STATISTICS Statement (Multistore Table) [Dynamic Tiering]

Drops data statistics objects.

Syntax
Caution This information applies only for the SAP HANA option cited in the title of this topic. You must have purchased the license for it in order to use the information contained in this topic.
Note While SAP HANA and dynamic tiering share much of the same base SQL syntax, there are some differences in supported and nonsupported clauses and parameters. The following syntax applies only to dynamic tiering.
 DROP STATISTICS { <data_statistics_name> 
   | ON <data_sources> [<source_storage_type>] [[HAVING] <match_properties>]}
Syntax Element
<data_statistics_name>

A unique name for the data statistics object to be dropped. 

<data_statistics_name> ::= [<schema_name>.]<identifier>

If you specify <data_statistics_name>, you cannot specify the ON <data_sources> clause at the same time.

<data_sources>
Data sources for the data statistics objects to be dropped. For TOPK, SIMPLE, and HISTOGRAM objects, specify multiple columns to drop each data statistics object built on the data source <table_name> <column_name>.
<data_sources> ::= <table_name> 
[(<column_name>{, <column_name>}...)]
<table_name>

Drops statistics for the specified table, with the optional schema name. 

<table_name> ::= [<schema_name>.]<identifier>

<schema_name> ::= <unicode_name>

For more information on schema names and identifiers, see Identifiers.

<column_name>

Drops statistics for the specified column.

<column_name> ::= <identifier>

If no <column_name> is specified, then all statistics for the table that match the specified <data_statistics_type> are dropped, including table-wide statistics (currently only RECORD COUNT).

<source_storage_type>
The storage type of table partitions affected by the statement. DEFAULT specifies partitions in the HANA column store; EXTENDED specifies partitions on extended storage. If no type is specified, the statement applies to partitions irrespective of their storage types. The storage type clause restricts the statement scope to table partitions of a specified storage type. You can implement different statistics types for different storage classes, and create statistics to address specific performance problems that only affect the query plan for one of the storage classes of the table. The statement returns an error if no partitions of the specified storage type exist. 
<source_storage_type> ::= FOR { DEFAULT | EXTENDED } STORAGE
<match_properties>

Specifies properties to use for matching the data statistics to drop.

<match_properties> ::= <match_property> [<...>] 

<match_property> ::=
 TYPE <data_statistics_type>
 | <refresh_type>

If TYPE is not specified, then all data statistics objects on specified data sources are dropped (ALL). See the CREATE STATISTICS Statement topic for descriptions of the supported data statistics types.

<data_statistics_type>

Specifies the type of data statistics objects to match when selecting the data statistics to drop. If TYPE is not specified, all data statistics objects on specified data sources are dropped.See CREATE STATISTICS Statement (Multistore Table) [Dynamic Tiering] for descriptions of the supported data statistics types.

<data_statistics_type> ::= 
 HISTOGRAM 
 | SIMPLE 
 | SKETCH 
 | RECORD COUNT 
 | TOPK
 | ALL
<refresh_type>

Specifies the refresh strategy to match when selecting the data statistics to drop. ALL is the default.

<refresh_type> ::= REFRESH TYPE {AUTO | MANUAL | ALL}
Example
Drop all statistics over T.col2 extended storage partitions.
DROP STATISTICS ON T(col2) FOR EXTENDED STORAGE
Drop a data statistics object with the name "HISTOGRAM_T_X".
DROP STATISTICS "HISTOGRAM_T_X"
Drop all statistics on table T, which drops any existing statistics over either partitions in the HANA column store or partitions in extended storage.
DROP STATISTICS on T
