ALTER SYSTEM UNINITIALIZE SERVICE Statement (Remote Data Sync)

Removes the system objects required by remote data sync server from the current tenant database.

Syntax
Caution This information applies only for the SAP HANA option cited in the title of this topic. You must have purchased the license for it in order to use the information contained in this topic.
Note While SAP HANA and remote data sync share much of the same base SQL syntax, there are some differences in supported and nonsupported clauses and parameters. The following syntax applies only to remote data sync.
ALTER SYSTEM UNINITIALIZE SERVICE 'rdsyncserver' 
[ AT [ LOCATION ] '<hostname>[:<port_number> ]']
Syntax Elements
<hostname> ::= <identifier>

This parameter specifies the name of the host where the remote data sync server is installed.

<port_number> ::= <identifier>

This parameter specifies the port number that the remote data sync server uses on the host.

Description

Execute this statement as the SYSTEM user in the tenant database.

This statement must be executed once for every remote data sync server associated with the tenant database. Once you execute this statement, the remote data sync server is in an uninitialized state and it cannot process synchronization requests.

When the last remote data sync server for a tenant is uninitialized, the remote data sync system objects are removed from the tenant.

Typically you execute an ALTER DATABASE...REMOVE statement in SYSTEMDB after executing an ALTER SYSTEM UNINITIALIZE SERVICE statement in the tenant database.

Example

The following example shows you how to remove the remote data sync system objects from a tenant database.

ALTER SYSTEM UNINITIALIZE SERVICE 'rdsyncserver'
