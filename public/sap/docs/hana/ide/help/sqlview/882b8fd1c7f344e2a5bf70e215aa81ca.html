M_TABLE_STATISTICS System View

Returns the table runtime statistics.

Structure

Column name

Data type

Description

SCHEMA_NAME

NVARCHAR(256)

Specifies the schema name to which the table belongs.

TABLE_NAME

NVARCHAR(256)

Specifies the table name to be counted.

INSERT_COUNT

BIGINT

Specifies the count of INSERT statements for the table.

DELETE_COUNT

BIGINT

Specifies the count of DELETE statements for the table.

UPDATE_COUNT

BIGINT

Specifies the count of UPDATE statements for the table.

REPLACE_COUNT

BIGINT

Specifies the count of REPLACE statements for the table.

MERGE_COUNT

BIGINT

Specifies the count of MERGE INTO statements for the table.

SELECT_COUNT

BIGINT

Specifies the count of SELECT statements for the table.

LAST_MODIFY_TIME

TIMESTAMP

Specifies the timestamp when the last INSERT, UPDATE, DELETE, or REPLACE statement was executed.

LAST_SELECT_TIME

TIMESTAMP

Specifies the timestamp when the last SELECT statement was executed.
Additional Notes

  • The M_TABLE_STATISTICS system view is not transactional; when a DML statement is rolled back, count values in the view remain unchanged.

  • DML statements that do not result in changes to data still increment the DML counters for the relevant tables.

  • DML-related counters and SELECT_COUNT are only applicable to user tables.

  • Generally, when a single SELECT statement touches more than one table (regardless of whether results are returned from those tables), the SELECT_COUNTER for each table touched is incremented. However, SELECT statements that are subqueries do not increment the SELECT_COUNT counter for the tables referenced in the subquery. For example, a SELECT statement that is a subquery of an INSERT statement does not increment the SELECT_COUNT counter.

  • Nested SELECT statements, for example in procedures, increment the SELECT counter, with one exception: for a SELECT statement in a calculation view, nested SELECT statements cannot increment the SELECT counter when nested SELECT statements are optimized.

This view has a resettable counterpart. You can see the values since the last reset in the dependent view M_TABLE_STATISTICS_RESET. To reset the view, execute this statementALTER SYSTEM RESET MONITORING VIEW SYS.M_TABLE_STATISTICS_RESET;.
