IMPORT SCAN Statement (Data Import Export)

Searches the specified path for exported objects.

Syntax
IMPORT SCAN <path>
Syntax Elements
<path>

Specifies the path of the directory to be scanned for import data.

<path> ::= <string_literal>
Description

The IMPORT SCAN statement searches the given path for objects exported with the EXPORT statement and stores its results in the following session-local temporary table at current_schema of the session. If current_schema of the session is invalid, then IMPORT SCAN throws an exception as it cannot create its result table of the current_schema.

<current_schema>.#IMPORT_SCAN_RESULT

Column Name

Data type

Description

SCHEMA_NAME

NVARCHAR(255)

The schema of the object

OBJECT_NAME

NVARCHAR(255)

The name of the object

OBJECT_TYPE

VARCHAR(16)

The type of the object (TABLE, VIEW, and so on)

EXISTS

TINYINT

Does the object already exist in the system? (0/1)

Example

Scan the path '/data/import_path' for import data.

IMPORT SCAN '/data/import_path'

Select the data in the #IMPORT_SCAN_RESULT table to get the result of the scan.

SELECT * FROM #IMPORT_SCAN_RESULT;
