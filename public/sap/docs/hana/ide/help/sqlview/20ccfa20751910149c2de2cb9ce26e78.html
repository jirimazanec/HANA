REORG_OVERVIEW System View

Provides an overview of landscape redistributions.

Structure

Column name

Data type

Description

REORG_ID

INTEGER

Specifies the unique identifier for the executed reorg plan

STATUS

VARCHAR(32)

Specifies the status of the plan execution. The status can be:

  • STARTED: The plan execution has started.

  • FINSHED: The plan exection finished successfully.

  • FAILED: The plan exection is treated as failed.

START_DATE

TIMESTAMP

Specifies the timestamp when the plan execution started

END_DATE

TIMESTAMP

Specifies the timestamp when plan execution stopped. This field remains empty until the exection is running. In case of reboots during plan execution, this field remains empty

USER

VARCHAR(256)

Specifies the user that executed the table redistribution

ALGORITHM_ID

INTEGER

Specifies the ID of the algorithm for the table redistribution

PARAMETERS

VARCHAR(1024)

Specifies the parameters given for the table redistribution call
Full Description

This view contains information about table redistribution executions on this system. Each call to REORG_EXECUTE is stored in this view. Execution duration and the status of each table redistribution on this system can be determined.

Only executed plan information is stored in this view. Calls of REORG_GENERATE without REORG_EXECUTE do not appear in this view.
