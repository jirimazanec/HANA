M_ES_LOCKS System View [Dynamic Tiering]

Locks held in SAP HANA dynamic tiering.

Caution This information applies only for the SAP HANA option cited in the title of this topic. You must have purchased the license for it in order to use the information contained in this topic.
Structure
Column name Data type Description
CONNECTION_NAME VARCHAR(255) The name of the current connection.
CONNECTION_ID BIGINT Connection id that has the lock. This connection id can join the REMOTE_CONNECTION_ID of M_REMOTE_STATEMENTS.
USER_ID VARCHAR(255) User id associated with this connection.
TABLE_TYPE VARCHAR(32) The type of table. This type is either 'BASE TABLE' for a table, 'GLOBAL TEMPORARY TABLE' for global temporary table, or 'MATERIALIZED VIEW' for a materialized view.
SCHEMA_NAME NVARCHAR(256) The schema name of the table.
TABLE_NAME NVARCHAR(256) Table on which the lock is held.
INDEX_ID BIGINT The index ID or NULL.
LOCK_CLASS VARCHAR(8) The lock class. One of SCHEMA, ROW, TABLE, or POSITION.
LOCK_DURATION_TYPE VARCHAR(16) The duration of the lock. One of TRANSACTION, POSITION, or CONNECTION.
LOCK_TYPE VARCHAR(10) Depending on the lock class, it could be one of:
  • SHARED, EXCLUSIVE for SCHEMA
  • READ, INTENT, READPK, WRITE, WRITENOPK, SURROGATE for ROW
  • SHARED, INTENT, EXCLUSIVE for TABLE
  • PHANTOM, INSERT for POSITION
ROW_IDENTIFIER BIGINT The identifier for the row the lock starts on, or NULL.
ROW_RANGE BIGINT The number of contiguous rows that are locked. Row locks in the DELTA store can either be a single row, or a range of rows.
