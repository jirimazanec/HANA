DROP STATISTICS Statement (Extended Store Table) [Dynamic Tiering]

Drops data statistics objects.

Syntax
Caution This information applies only for the SAP HANA option cited in the title of this topic. You must have purchased the license for it in order to use the information contained in this topic.
Note While SAP HANA and dynamic tiering share much of the same base SQL syntax, there are some differences in supported and nonsupported clauses and parameters. The following syntax applies only to dynamic tiering.
 DROP STATISTICS {<data_statistics_name> 
| ON <data_sources> [[HAVING] <match_properties>] [TYPE <data_statistics_type>] [REFRESH <refresh_type>]}
Syntax Element
<data_statistics_name>

Specifies the name of a data statistics object. 

<data_statistics_name> ::= [<schema_name>.]<identifier>

If you specify <data_statistics_name>, you cannot specify the ON <data_sources> clause at the same time.

<data_sources>
Data sources for the data statistics objects to be dropped. 
<data_sources> ::= <table_name> 
[(<column_name>{, <column_name>}...)] [<match_type>]
<table_name>

Drops statistics for the specified table, with the optional schema name. 

<table_name> ::= [<schema_name>.]<identifier>

<schema_name> ::= <unicode_name>

For more information on schema names and identifiers, see Identifiers.

<column_name>

Drops statistics for the specified column.

<column_name> ::= <identifier>

If no <column_name> is specified, then all statistics for the table that match the specified <data_statistics_type> are dropped, including table-wide statistics (currently only RECORD COUNT).

<match_type>

Controls which data statistics objects to match to <data_sources>.

<match_type> ::= EXACT | CASCADE

If <match_type> is not specified, then all data statistics object(s) that reference all or some of the columns specified in <data_sources> are dropped.

Specify EXACT to drop a data statistics object that precisely matches <data_sources> (including column order).

Specify CASCADE to drop data statistics objects that reference at least one column in <data_sources>.

Use this table to understand how matching is performed based on <match_type> when <data_sources> is T(A, B, C):

<data_statistics_type>
Identifies which data statistics objects are dropped. If TYPE is not specified, all data statistics objects on specified data sources are dropped.
<data_statistics_type>

<data_statistics_type> ::= 
HISTOGRAM 
   | SIMPLE
   | SKETCH
   | RECORD COUNT
   | TOPK 
   | ALL
The data statistics type identifies which data statistics objects are to be dropped. If TYPE is not specified, all data statistics objects on specified data sources are dropped. See CREATE STATISTICS Statement (Extended Store Table) [Dynamic Tiering] for descriptions of the supported data statistics types.
<refresh_type>

Specifies the refresh strategy to match when selecting the data statistics to drop. ALL is the default.

<refresh_type> ::= {AUTO | MANUAL | ALL}
Example
You drop a data statistics object with the name "HISTOGRAM_T_X".
DROP STATISTICS "HISTOGRAM_T_X"
