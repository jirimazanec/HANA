TABLE_COLUMNS System View

Provides information about available table columns.

Structure

Column name

Data type

Description

SCHEMA_NAME

NVARCHAR(256)

The schema name

TABLE_NAME

NVARCHAR(256)

The table name

TABLE_OID

BIGINT

The object ID of the table

COLUMN_NAME

NVARCHAR(256)

The name of the column

POSITION

INTEGER

The ordinal position of the column in a record

DATA_TYPE_ID

SMALLINT

The SQL data type ID of the column

DATA_TYPE_NAME

VARCHAR(16)

The SQL data type name of the column

OFFSET

SMALLINT

The offset of the column in the record

LENGTH

INTEGER

The number of chars for char types, number of max digits for numeric types; number of chars for datetime types, and number of bytes for LOB types

SCALE

INTEGER

The numeric types: the maximum number of digits to the right of the decimal point; time, timestamp: the decimal digits are defined as the number of digits to the right of the decimal point in the second's component of the data

IS_NULLABLE

VARCHAR(5)

Specifies whether the column is allowed to accept null value: TRUE or FALSE

DEFAULT_VALUE

NVARCHAR(5000)

The default value of the column

COLLATION

NVARCHAR(256)

The collation used for this column

COMMENTS

NVARCHAR(5000)

The description for this column

MAX_VALUE

VARCHAR(1)

Deprecated: will not be set any more

MIN_VALUE

VARCHAR(1)

Deprecated: will not be set any more

CS_DATA_TYPE_ID

INTEGER

The column store data type ID

CS_DATA_TYPE_NAME

VARCHAR(16)

The column store data type name

DDIC_DATA_TYPE_ID

INTEGER

The DDIC data type ID

DDIC_DATA_TYPE_NAME

VARCHAR(7)

The DDIC data type name

COMPRESSION_TYPE

VARCHAR(9)

The type of compression: DEFAULT, PREFIXED, SPARSE, CLUSTERED, SPARSE, CLUSTERED, INDIRECT, RLE, and LINEAR RLE. The default value for the column is shown. The actual value can be checked in the system view M_ CS_COLUMNS. For row store columns NONE is shown.

INDEX_TYPE

VARCHAR(4)

The type of index: NONE or FULL

COLUMN_ID

BIGINT

The ID of the column

PRELOAD

VARCHAR(5)

Specifies if the column is preloaded: TRUE or 'FALSE. Preload information can be NULL in case of row store, virtual columns or tables on extended storage.

GENERATED_ALWAYS_AS

NVARCHAR(1000)

Specifies the expression of the column created by GENERATED... AS

HAS_SCHEMA_FLEXIBILITY

VARCHAR(5)

Specifies if the column has schema flexibility: TRUE or FALSE

FUZZY_SEARCH_INDEX

VARCHAR(5)

Specifies if the column has fuzzy search index: TRUE or FALSE

FUZZY_SEARCH_MODE

VARCHAR(16)

Specifies fuzzy search mode

MEMORY_THRESHOLD

INTEGER

Specifies memory threshold in bytes for LOB types

LOAD_UNIT

VARCHAR(6)

Specifies the unit of column data loading: TABLE, COLUMN, or PAGE

GENERATION_TYPE

VARCHAR(22)

Specifies ALWAYS AS if the column is generation column, ALWAYS AS IDENTITY, BY DEFAULT AS IDENTITY if the column is an identity column whose values are always generated or generated by default, and NULL otherwise

IS_CACHABLE

VARCHAR(5)

Specifies whether the column is part of cached data: TRUE or FALSE

IS_CACHE_KEY

VARCHAR(5)

Specifies whether the column is part of cache key: TRUE or FALSE

ROW_ORDER_POSITION

SMALLINT

Specifies the row order position
