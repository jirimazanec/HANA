IS_SQL_INJECTION_SAFE Function (Security)

Checks a specified SQL identifier for possible SQL injection risks.

Syntax
IS_SQL_INJECTION_SAFE(<value>[, <max_tokens>])
Syntax Elements
<value>

Specifies the string to be checked.

<value> ::= <string>
<max_tokens>

Specifies the maximum number of tokens expected in <value>. The default value is 1.

<max_tokens> ::= <integer>
Description

IS_SQL_INJECTION_SAFE can be used to ensure a string contains a defined number of tokens before using it in a SQL command.

IS_SQL_INJECTION_SAFE returns 1 (safe) if the number of tokens found does not exceed <max_tokens>; otherwise, the function returns 0 (not safe).

Example

The following query returns 0 because two tokens were found and the default to check for was 1.

SELECT IS_SQL_INJECTION_SAFE('tab,le') "safe" FROM DUMMY;

The following query returns 1 because the number of tokens found did not exceed <max_tokens>:

SELECT IS_SQL_INJECTION_SAFE('CREATE STRUCTURED PRIVILEGE', 3) "safe" FROM DUMMY;

The following query returns 0 because comments are not allowed.

SELECT IS_SQL_INJECTION_SAFE('mytab /*', 4) "safe" FROM DUMMY;
