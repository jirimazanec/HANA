M_JOBEXECUTORS System View

Provides job executor statistics.

Structure

Column name

Data type

Unit

Description

HOST

VARCHAR(64)

  

Host name

PORT

INTEGER

  

Internal port

MAX_CONCURRENCY

INTEGER

Counter

Maximum number of concurrently running worker threads (config).

The job executor usually keeps MAX_CONCURRENCY worker threads busy. As a result more worker threads are started if some worker threads are in wait state for some time. Even if these additional worker threads are terminated, some of them are kept for future use and counted in PARKED_WORKER_COUNT.

TOTAL_WORKER_COUNT

INTEGER

Counter

Total number of worker threads, including one extra worker thread for emergency diagnostic purposes.

PARKED_WORKER_COUNT

INTEGER

Counter

Number of parked worker threads

FREE_WORKER_COUNT

INTEGER

Counter

Total number of worker threads that are idle and may immediately take up a job to work on

SYS_WAITING_WORKER_COUNT

INTEGER

Counter

Number of worker threads waiting in system. A worker thread may wait for any other kind of synchronization.

JOB_WAITING_WORKER_COUNT

INTEGER

Counter

Number of worker threads waiting for a job on another worker thread

YIELD_WAITING_WORKER_COUNT

INTEGER

Counter

Number of worker threads waiting on yield. Sometimes a job running on a worker thread yields execution to another job that is more important,

TOTAL_WAITING_JOB_COUNT

BIGINT

Counter

Number of all jobs waiting for execution

QUEUED_WAITING_JOB_COUNT

BIGINT

Counter

Number of jobs queued for execution

WORKER_CREATE_COUNT

BIGINT

Counter

The number of threads created

This view has a resettable counterpart. This means, you can also see the values since the last reset in the dependent view M_JOBEXECUTORS_RESET. To reset the values, use the SQL command: ALTER SYSTEM RESET MONITORING VIEW SYS.M_JOBEXECUTORS_RESET or the management console command: monitor reset -n M_JOBEXECUTORS_RESET.
