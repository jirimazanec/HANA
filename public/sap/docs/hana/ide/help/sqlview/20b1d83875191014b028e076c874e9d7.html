M_LANDSCAPE_HOST_CONFIGURATION System View

Specifies the host roles in a distributed landscape.

Structure

Column name

Data type

Description

HOST

VARCHAR(64)

The host name.

HOST_ACTIVE

VARCHAR(128)

The host active status (a summary of the active values of all the services on that host).

HOST_STATUS

VARCHAR(128)

The host status.

OK

The host is operational; the host actual role equals the configured role.

IGNORE

The host is operational; the host configured as standby is available, but not used.

INFO

The host is operational; the host actual role is different from the configured role.

WARNING

The host is not operational; the host should become operational after startup/failover.

ERROR

The host not operational; the host is in an erroneous state.

FAILOVER_STATUS

VARCHAR(128)

The failover status.

(empty value)

No failover is pending or active.

waiting .. sec

A failure is detected. Waiting is required to prevent an unnecessary failover. A restart of another host does not trigger an immediate failover.

waiting

A failure is detected. Waiting for another starting host.

failover to 'host'

Failover to an active host named 'host'.

failback to 'host'

Failback to worker host active. This happens when a standby host was assigned and is stopped. There is no automatic failback while the standby host is assigned because this would cause downtime.

failed

The failover failed. See nameserver.trc for details

FAILOVER_GROUP

VARCHAR(256)

DEPRECATED: replaced by FAILOVER_..._GROUP columns.

FAILOVER_CONFIG_GROUP

VARCHAR(256)

The configured failover group.

Hosts can be grouped. During failover, a host is looked for within the same group. If no host is available in the same group, hosts from other groups are used.

FAILOVER_ACTUAL_GROUP

VARCHAR(256)

The actual failover group.

NAMESERVER_CONFIG_ROLE

VARCHAR(16)

The nameserver configured roles: MASTER, SLAVE. During installation, up to 3 hosts are automatically configured as MASTER candidates. All others are SLAVE.

NAMESERVER_ACTUAL_ROLE

VARCHAR(16)

The nameserver actual roles: MASTER, SLAVE. Exactly one of the master candidates is MASTER, all others are SLAVE.

The actual MASTER name server and MASTER index server are both on the same host.

INDEXSERVER_CONFIG_ROLE

VARCHAR(16)

The indexserver configured roles: WORKER, STANDBY.

During SAP HANA system installation or host addition, hosts can be assigned the SAP HANA server roles WORKER or STANDBY. The host can also be assigned other roles.

INDEXSERVER_ACTUAL_ROLE

VARCHAR(16)

The indexserver actual roles: MASTER, SLAVE, STANDBY. Exactly one host is MASTER, all others are SLAVE or STANDBY.

HOST_CONFIG_ROLES

VARCHAR(64)

Specifies the host's configured database and SAP HANA option roles. The host configured roles are: ETS_WORKER, ETS_STANDBY, EXTENDED_STORAGE_WORKER, EXTENDED_STORAGE_STANDBY, STANDBY, STREAMING, WORKER, XS_WORKER, XS_STANDBY.

HOST_ACTUAL_ROLES

VARCHAR(64)

Specifies the host's current database and SAP HANA option roles (these may change due to Host Auto-Failover). The host actual roles are: ETS_WORKER, ETS_STANDBY, EXTENDED_STORAGE_WORKER, EXTENDED_STORAGE_STANDBY, STANDBY, STREAMING, WORKER, XS_WORKER, XS_STANDBY.

STORAGE_PARTITION

-

DEPRECATED: replaced by the STORAGE_CONFIG_PARTITION, STORAGE_ACTUAL_PARTITION columns.

STORAGE_CONFIG_PARTITION

 INTEGER

The stable subpath to reassign the same storage partition after failovers.

STORAGE_ACTUAL_PARTITION

INTEGER

The unique number of the mount sub-directory used by the host for storing data and logs (also known as "subpath").

WORKER_CONFIG_GROUPS

VARCHAR(256)

The stable classification values to assign hosts to logical worker groups.

WORKER_ACTUAL_GROUPS

VARCHAR(256)

The current classification values to assign hosts to logical worker groups.

REMOVE_STATUS

VARCHAR(16)

Removes progress.
More information

You can also find information in the SAP HANA Administration Guide for configuring clients for failover support.
