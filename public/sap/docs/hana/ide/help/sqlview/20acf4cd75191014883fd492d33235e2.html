Full Description

Column Index Types

FULL means a full inverted index, which is a data structure that assigns each value in the column a list of its positions. As of SPS 09 on Intel architecture and the initial shipment stack on Power architecture, it can be used in combination with DEFAULT/UNCOMPRESSED column or PAGED columns. The additional memory consumption of FULL indexes is typically quite high as the more distinct the column is, the higher the memory consumption of the index. On the other hand, having such an index means, that reverse look-ups on the indexed column are usually very fast.

BLOCK means a block-based index type and can be used with ‘compressed’ columns, that is, ones that have a compression type of PREFIX, SPARSE, RLE, LRLE, INDIRECT and CLUSTED. ‘Block-based’ means that each value is only assigned a list of fixed-size blocks in the column that contain it. This means that the index is typically much smaller than a corresponding full inverted index, but also that reverse look-ups are effectively a combination of reading the index (fast) and scanning the read blocks (possibly slow, depending on the compression of the column).

Note In general, it is not recommended to change the different index types. The database is able to decide the best index type for a given column.
