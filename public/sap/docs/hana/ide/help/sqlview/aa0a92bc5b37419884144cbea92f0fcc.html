EFFECTIVE_MASK_EXPRESSIONS System View

Provides information as to how data is exposed to certain users in terms of data masking.

Structure

Column name

Data type

Description

ROOT_SCHEMA_NAME

NVARCHAR(256)

Specifies the schema name. An equal predicate is required.

ROOT_OBJECT_NAME

NVARCHAR(256)

Specifies the name of the object. An equal predicate is required.

ROOT_COLUMN_NAME

NVARCHAR(256)

Specifies the column name. An equal predicate is required.

USER_NAME

NVARCHAR(256)

Specifies the user name. An equal predicate is required.

EFFECTIVE_USER_NAME

NVARCHAR(256)

Specifies the name of the effective user whom the mask expression enforced on the underlying object it applies to.

DEPENDENT_SCHEMA_NAME

NVARCHAR(256)

Specifies the schema name of the dependent object.

DEPENDENT_OBJECT_NAME

NVARCHAR(256)

Specifies the object name of the dependent object.

DEPENDENT_OBJECT_TYPE

VARCHAR(32)

Specifies the type name of the dependent object.

DEPENDENT_COLUMN_NAME

NVARCHAR(256)

Specifies the name of the dependent column.

UNDERLYING_SCHEMA_NAME

NVARCHAR(256)

Specifies the schema name of the object at its current level.

UNDERLYING_OBJECT_NAME

NVARCHAR(256)

Specifies the object name of the object at its current level.

UNDERLYING_OBJECT_TYPE

VARCHAR(32)

Specifies the type name of the object at its current level.

UNDERLYING_COLUMN_NAME

NVARCHAR(256)

Specifies the name of the column at its current level.

MASK_EXPRESSION

NVARCHAR(5000)

Specifies the mask expression string.

MASK_EXPRESSION_STATUS

VARCHAR(32)

Specifies whether the masking is applied or not. This can be specified as: NULL, APPLIED, or NOT APPLIED. The status is NOT APPLIED if you have an UNMASKED privilege. The status is NULL if the specified column does not have a mask definition.
