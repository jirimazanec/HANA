ALTER VIRTUAL TABLE Statement (Data Definition)

Modifies a virtual table's column properties, and lets you refresh the metadata of a virtual table.

Syntax
ALTER VIRTUAL TABLE <table_name>
 SET PROPERTY 'name' = 'value'[, 'name2' = 'value2', ...]
 | <table_name> UNSET PROPERTY 'name'[, 'name2', ...]
 | <table_name> ALTER <column_name> SET PROPERTY 'name' = 'value' [, 'name2' = 'value2', ...]
 | <table_name> ALTER <column_name> UNSET PROPERTY 'name'[, 'name2', ...]
 | REFRESH DEFINITION
Syntax Elements
<table_name>

Specifies a virtual table.

<table_name> ::= [<schema_name>.]<identifier>

<schema_name> ::= <unicode_name>
<column_name>

Specifies a virtual column.

<column_name> ::= <identifier>
REFRESH DEFINITION
Updates a virtual table to reflect metadata changes in the corresponding remote table.
Description

Any properties that are not set by using the ALTER VIRTUAL TABLE statement are marked as read-only, which cannot be updated by the ALTER VIRTUAL TABLE statement.

Use the DROP TABLE <table_name> to drop a virtual table.

Spatial data types are not supported in virtual tables.

Examples

Create a virtual table on an existing remote source. 

CREATE REMOTE SOURCE S ADAPTER ASEODBC CONFIGURATION 'DSN=ProductionASE' 
   WITH CREDENTIAL TYPE 'password' USING 'user="aseuser";password="asepassword"';
CREATE VIRTUAL TABLE VT AT S.db_name.owner.table_name;

Set a new property for the virtual table VT, then update and unset it.

ALTER VIRTUAL TABLE VT SET PROPERTY 'name1' = 'value1';
ALTER VIRTUAL TABLE VT SET PROPERTY 'name1' = 'value2';
ALTER VIRTUAL TABLE VT UNSET PROPERTY 'name1';

Set a new property for the virtual column A in virtual table VT, then update and unset it.

ALTER VIRTUAL TABLE VT ALTER A SET PROPERTY 'name1' = 'value1';
ALTER VIRTUAL TABLE VT ALTER A SET PROPERTY 'name1' = 'value2';
ALTER VIRTUAL TABLE VT ALTER A UNSET PROPERTY 'name1';
