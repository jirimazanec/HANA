ALTER SYSTEM INITIALIZE SERVICE Statement (Remote Data Sync)

Creates the system objects required by the remote data sync server in the current tenant so that the remote data sync server can accept synchronization requests.

Caution This information applies only for the SAP HANA option cited in the title of this topic. You must have purchased the license for it in order to use the information contained in this topic.
Note While SAP HANA and remote data sync share much of the same base SQL syntax, there are some differences in supported and nonsupported clauses and parameters. The following syntax applies only to remote data sync.
Syntax
ALTER SYSTEM INITIALIZE SERVICE 'rdsyncserver' 
[ AT [ LOCATION ] '<hostname>[:<port_number> ]']
Syntax Elements
<hostname> ::= <identifier>

This parameter specifies the name of the host where the remote data sync server is installed.

<port_number> ::= <identifier>

This parameter specifies the port number that the remote data sync server uses on the host.

Description

Execute this statement as the SYSTEM user in the tenant database.

Before executing this statement, you must first execute an ALTER DATABASE...ADD statement as the SYSTEM user in SYSTEMDB.

Execute this statement once for every remote data sync host associated with the tenant. After this statement executes successfully, the remote data sync server restarts and can accept synchronization requests.

Example

This example initializes the rdsyncserver service on host myhostname, port 30027. 00 in the port number is the instance number of the system.

ALTER SYSTEM INITIALIZE SERVICE 'rdsyncserver' at 'myhostname:30027'
