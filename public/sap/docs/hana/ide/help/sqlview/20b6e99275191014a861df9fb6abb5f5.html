M_PASSWORD_POLICY System View

Defines effective password policy settings.

Structure

Column name

Data type

Description

PROPERTY

VARCHAR(128)

Specifies the configuration property. The following describes the contents of the individual lines of the monitor view (VALUE), as identified by PROPERTY:

force_first_password_change
Specifies if a user is forced to change their administrator-given password before being allowed to work any further.
last_used_passwords
Specifies the number of recently used passwords of a user, which they cannot reuse.
maximum_invalid_connect_attempts
Specifies the maximum number of allowed invalid connect attempts before a user is locked out.
maximum_password_lifetime
Specifies the number of days that a password stays valid.
maximum_unused_initial_password_lifetime
Specifies the number of days that an unused administrator-given password stays valid.
Note

In SAP HANA 1.0 SP 12 and earlier, the parameter name is "maximum_unused_inital_password_lifetime" (note the missing 'i' in 'initial'). Use of the previous spelling is supported, but it is recommended that you update your configuration to use the correct spelling.

maximum_unused_productive_password_lifetime
Specifies the number of days that an unused user-given password stays valid.
minimal_password_length
Specifies the minimum number of characters required by a valid password.
minimum_password_lifetime
Specifies the minimum number of days before a user-given password can be changed by that user.
password_expire_warning_time
Specifies that warnings about nearly expired passwords are shown at the configured number of days before expiration.
password_layout
Describes the kind(s) of characters that the password must consist of.
password_lock_time
Specifies the number of minutes the user is locked out after too many invalid connect attempts.

VALUE

VARCHAR(128)

Specifies the value.
Full description

This view contains several lines of information about the current password policy.
