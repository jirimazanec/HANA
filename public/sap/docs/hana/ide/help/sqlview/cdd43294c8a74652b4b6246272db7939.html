PARTITIONED_TABLES System View

General partitioning information for all partitions of a table.

You need the DATA ADMIN system privilege see information for tables owned by other users, unless you have SELECT permission on those tables.

If your system is licensed for the dynamic tiering option, this monitoring view shows information for the extended storage corresponding to the connected HANA database.

Structure
Column Name Data Type Description
SCHEMA_NAME NVARCHAR(256) Schema name.
TABLE_NAME NVARCHAR(256) Table name.
LEVEL_1_EXPRESSION NVARCHAR(5000) Partitioning expression for first-level partitions.
LEVEL_1_COUNT INTEGER Total of partitions at the first level.
LEVEL_1_TYPE VARCHAR(16) The partitioning type at the first level. Valid values are: 'HASH'/ 'RANGE'/'ROUNDROBIN'/'REPLICATE'.
LEVEL_2_EXPRESSION NVARCHAR(5000) Partitioning expression for second-level partitions.
LEVEL_2_COUNT INTEGER Total of partitions at the second level.
LEVEL_2_TYPE VARCHAR(16) The partitioning type at the second level. Valid values are: 'HASH' and 'RANGE'
CROSS_STORAGE_

UNIQUE_CONSTRAINTS

 VARCHAR(5) Valid values are: ''TRUE' or 'FALSE'. Specify 'TRUE' when constraint checking is done across stores.
PARTITIONING_ON_NON_

PRIMARY_KEY_COLUMNS

 VARCHAR(5) Valid values are: ''TRUE' or 'FALSE'. Specify 'TRUE' when partitioning is on a non-primary key.
DYNAMIC_RANGE_THRESHOLD BIGINT Threshold after which a new partition is created dynamically.
EXTENDED_STORAGE_ENABLE_DELTA VARCHAR(5) Valid values are: ''TRUE', 'FALSE', or 'NULL'. 'TRUE' means that partitions in extended storage use a delta store. 'NULL' means that the system couldn't retrieve status from extended storage.
