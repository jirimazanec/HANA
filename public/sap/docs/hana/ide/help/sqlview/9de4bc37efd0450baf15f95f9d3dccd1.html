HIERARCHY_LEVELED Function (Hierarchy)

Creates a hierarchy based on source data that has a leveled format.

Syntax
HIERARCHY_LEVELED (
 <hierarchy_genfunc_source_spec>
 [ <hierarchy_genfunc_level_spec> ]
 [ <hierarchy_genfunc_cache_spec> ] )
Syntax Elements
<hierarchy_genfunc_source_spec>

Specifies the source table for the hierarchy to be generated from where the source has the following characteristics:

  • each source column defines either a level or a level attribute of the generated hierarchy

  • each row defines one path from a root node to a leaf node

  • the sequence of source columns determines their level; the sort order of rows determines the sorting/relative preorder RANK of sibling nodes

<hierarchy_genfunc_level_spec>

Specifies which source columns define a level.

<hierarchy_genfunc_level_spec> ::= LEVELS <column_list_clause>

<column_list_clause> ::= ( <column_name>, ... ) 

<column_name> ::= <identifier>

If <hierarchy_genfunc_level_spec> is not specified, then all source columns are treated as level columns defining a level.

If <column_name> is not a member of the projection list of <hierarchy_genfunc_source_spec>, then an error is returned.

<hierarchy_genfunc_cache_spec>

Specifies the caching policy for the generated hierarchy.

<hierarchy_genfunc_cache_spec> ::= 
 CACHE 
 | NO CACHE 
 | CACHE FORCE

Caching may improve the performance for subsequent navigations on the same hierarchy.

CACHE

The generated hierarchy is cached if the system assesses the source to be reliably deterministic. This is the default behavior.

NO CACHE

The generated hierarchy is not cached.

CACHE FORCE

The generated hierarchy is cached even if the source cannot be assessed to be reliably deterministic.

Description

HIERARCHY_LEVELED creates a hierarchy based on source data having a leveled format. The optional LEVELS specification adds the possibility of distinguishing between source attributes that define a new level, and additional custom level attributes.

In addition to all output columns of the HIERARCHY function, HIERARCHY_LEVELED also generates the following columns:

Attribute Name

Data Type

PARENT_ID

inferred from source level columns

NODE_ID

inferred from source-level columns and NOT NULL

HIERARCHY_LEVEL_NAME

NVARCHAR(256) NOT NULL
<source_column_name_1> <source_column_type_1>
<source_column_name_x> <source_column_type_x>
Examples

The following statement creates a column table and populates it with hierarchy data:

CREATE COLUMN TABLE h_mini_leveled_src ( level_a VARCHAR(2), level_b VARCHAR(2), level_c VARCHAR(2), level_d VARCHAR(2) );			
INSERT INTO h_mini_leveled_src VALUES ( 'A1', 'B1', 'C1', NULL );
INSERT INTO h_mini_leveled_src VALUES ( 'A1', 'B1', 'C2', NULL );
INSERT INTO h_mini_leveled_src VALUES ( 'A1', 'B2', 'C3', 'D1' );
INSERT INTO h_mini_leveled_src VALUES ( 'A1', 'B2', 'C3', 'D2' );
INSERT INTO h_mini_leveled_src VALUES ( 'A1', 'B2', 'C4', 'D3' );

The following example calls the HIERARCHY_LEVELED function on data selected from the h_mini_leveled_src table:

SELECT HIERARCHY_RANK AS Rank,
   HIERARCHY_TREE_SIZE AS TreeSize,
   HIERARCHY_PARENT_RANK AS P_Rank,
   HIERARCHY_LEVEL AS Level,
   HIERARCHY_IS_CYCLE AS Cycle,
   HIERARCHY_IS_ORPHAN AS Orphan,
   PARENT_ID AS Parent,
   NODE_ID AS Node,
   HIERARCHY_LEVEL_NAME AS LevelName,
   LEVEL_A AS A,
   LEVEL_B AS B,
   LEVEL_C AS C,
   LEVEL_D AS D
 FROM HIERARCHY_LEVELED(SOURCE h_mini_leveled_src)
 ORDER BY HIERARCHY_RANK;

The following example shows the use of the LEVELS clause to level the hierarchy to two levels (level_b and level_c):

SELECT HIERARCHY_RANK AS Rank,
   HIERARCHY_TREE_SIZE AS TreeSize,
   HIERARCHY_PARENT_RANK AS P_Rank,
   HIERARCHY_LEVEL AS Level,
   HIERARCHY_IS_CYCLE AS Cycle,
   HIERARCHY_IS_ORPHAN AS Orphan,
   PARENT_ID AS Parent,
   NODE_ID AS Node,
   HIERARCHY_LEVEL_NAME AS LevelName,
   LEVEL_A AS A,
   LEVEL_B AS B,
   LEVEL_C AS C,
   LEVEL_D AS D
 FROM HIERARCHY_LEVELED(SOURCE h_mini_leveled_src LEVELS (level_b, level_c) CACHE FORCE);
