M_CS_TABLES System View

Provides runtime data for column tables.

Structure

Column name

Data type

Unit

Description

HOST

VARCHAR(64)

  

Specifies the host name.

PORT

INTEGER

  

Specifies the internal port.

SCHEMA_NAME

NVARCHAR(256)

  

Specifies the schema name.

TABLE_NAME

NVARCHAR(256)

  

Specifies the table name.

PART_ID

INTEGER

  

Specifies the partition ID. Returns the following:

  • For partitioned tables, the part ID is equal to the sequential number of the partition, starting at 1.
  • In the case of replicated tables, the part ID is 1 for the original table and subsequent part IDs are assigned to replica tables.
  • The part ID is 0 for tables that are not partitioned.
  • A part ID value of -1 indicates that a modfication of the table schema is currently in progress.

MEMORY_SIZE_IN_TOTAL

BIGINT

Byte

Specifies that the total memory size is the sum of memory size in the main, delta, and history parts.

MEMORY_SIZE_IN_MAIN

BIGINT

Byte

Specifies the current memory consumption in main. This value varies depending on the number of attributes actually loaded and includes data for open transactions.

MEMORY_SIZE_IN_DELTA

BIGINT

Byte

Specifies the current memory consumption in delta.

MEMORY_SIZE_IN_HISTORY_MAIN

BIGINT

Byte

Specifies the current memory consumption in history-main. This value is 0 for normal, non-history tables.

MEMORY_SIZE_IN_HISTORY_DELTA

BIGINT

Byte

Specifies the current memory consumption in history-delta. This value is 0 for normal non-history tables.

ESTIMATED_MAX_MEMORY_SIZE_IN_TOTAL

BIGINT

Byte

Specifies the estimated maximum memory consumption, in total, for the fully loaded table (data for open transactions is not included).

LAST_ESTIMATED_MEMORY_SIZE

BIGINT

Byte

Specifies the last estimated memory consumption for the fully loaded table.

LAST_ESTIMATED_MEMORY_SIZE_TIME

TIMESTAMP

Specifies the last time the last estimated memory consumption was calculated.

RECORD_COUNT

BIGINT

Byte

Specifies the record count.

RAW_RECORD_COUNT_IN_MAIN

BIGINT

  

Specifies the current number of entries in the main part of the table. This value differs from the number of visible table main rows because there are entries of modified rows marked as invalidated.

RAW_RECORD_COUNT_IN_DELTA

BIGINT

  

Specifies the current number of entries in the table delta part. This value differs from the number of visible table delta rows because there are additional entries, such as deleted rows or updated rows. This column can contain deleted records.

RAW_RECORD_COUNT_IN_HISTORY_MAIN

BIGINT

  

Specifies the raw record count in history-main.

RAW_RECORD_COUNT_IN_HISTORY_DELTA

BIGINT

  

Specifies the raw record count in history-delta.

LAST_COMPRESSED_RECORD_COUNT

BIGINT

  

Specifies the number of entries in main during the last optimize compression run.

MAX_UDIV

BIGINT

  

Specifies the maximum table row number. This number is purely technical and only used internally.

MAX_MERGE_CID

BIGINT

  

Specifies the maximum commit-ID of transactions for which changes were already merged into the main table.

MAX_ROWID

BIGINT

  

Specifies the maximum row ID. This number is purely technical and only used internally.

IS_DELTA2_ACTIVE

VARCHAR(5)

Boolean

Specifies the flag to indicate that a second delta is used: TRUE, FALSE. During a table delta merge, updates and inserts are stored to a second delta because the first delta is locked.

IS_DELTA_LOADED

VARCHAR(5)

Boolean

Specifies the flag to indicate that the delta part of table is loaded: TRUE, FALSE.

IS_LOG_DELTA

VARCHAR(5)

Boolean

Specifies the flag to indicate that currently the delta log is being written: TRUE, FALSE.

PERSISTENT_MERGE

VARCHAR(5)

Boolean

Specifies the flag to indicate that the new main part will be written to the disk during a table delta merge, unless requested differently: TRUE, FALSE.

CREATE_TIME

TIMESTAMP

  

Specifies the creation time.

MODIFY_TIME

TIMESTAMP

  

Specifies the latest modify timestamp of any column store table run time data. This is used to trigger invalidation of related translation tables used by the column store join operations.

LAST_MERGE_TIME

TIMESTAMP

  

Specifies the point in time, in Unix time format, of the last time the table delta part was merged into the main part.

LAST_REPLAY_LOG_TIME

TIMESTAMP

  

Specifies the point in time, in Unix time format, of the last time the table log was replayed.

LAST_TRUNCATION_TIME

TIMESTAMP

Specifies the last time the table was truncated.

LAST_CONSISTENCY_CHECK_TIME

TIMESTAMP

Specifies the last time the table consistency was checked with the CHECK_TABLE_CONSISTENCY procedure.

LAST_CONSISTENCY_CHECK_ERROR_COUNT

BIGINT

Specifies the number of errors found in the last table consistency check.

LOADED

VARCHAR(10)

  

Specifies the flag to show how many columns of the table are loaded in memory (NO, PARTIALLY, and FULL). See M_CS_COLUMNS for each column.

READ_COUNT

BIGINT

  

Specifies the number of read accesses on the table or partition.

Note

This is not the number of SELECT statements against this table. A SELECT statement may involve several read accesses.

WRITE_COUNT

BIGINT

  

Specifies the number of write accesses on the table or partition.

Note

This is not the number of DML & DDL statements against this table. A DML or DDL statement may involve several write accesses.

MERGE_COUNT

BIGINT

  

Specifies the number of delta merges done on the table or partition.

IS_REPLICA

VARCHAR(5)

Boolean

Specifies the flag to indicate that the part is a replica.

UNUSED_RETENTION_PERIOD

INTEGER

  

Specifies the unused retention period.
