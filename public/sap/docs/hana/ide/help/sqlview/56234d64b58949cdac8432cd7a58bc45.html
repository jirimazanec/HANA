Global System Properties (global.ini)

The global.ini file stores global configuration properties for each service in the landscape.

You can add or modify global.ini parameters at the system or database level, but you must make all changes from the SYSTEMDB.

These are the global properties most frequently used for extended storage configuration:

Name Section Description Value Default
basepath_databackup persistence Default path for file-based data backups. File path ($DIR_INSTANCE)/backup/data
basepath_datavolumes_es persistence Default path to the dynamic tiering data directory (data_es). This path must be available to the dynamic tiering host, but need not be available to HANA hosts. Avoid including the default installation directory in the path. File path /HANA/data_es/<SID>
basepath_logbackup persistence Default path for log backup files. Log backups for dynamic tiering store only the transaction log and metadata releated to data modifications to the log backup files. File path ($DIR_INSTANCE)/backup/log
basepath_logvolumes_es persistence Default path to the dynamic tiering log directory (log_es). This path must be available to the dynamic tiering host, but need not be available to HANA hosts. Avoid including the default installation directory in the path. File path /HANA/log_es/<SID>
es_data_backup_buffer_size backup Amount of data written or read to (or from) the backup medium during extended storage backup or recovery in a single system call. A smaller value may cause too many system calls, slowing read/write performance on the backup subsystem. A larger value may burden the memory requirements from the dynamic tiering service, and the backup could fail due to lack of memory. 4 â€“ 32 8 MB
