M_CONTEXT_MEMORY System View

Provides memory allocator statistics.

Full description

This view has a resettable counterpart. This means, you can also see the values since the last reset in the dependent view M_CONTEXT_MEMORY_RESET. To reset the values, use the SQL command: ALTER SYSTEM RESET MONITORING VIEW SYS.M_CONTEXT_MEMORY_RESET or the management console command: monitor reset -n M_CONTEXT_MEMORY_RESET

This view contains information about memory consumption grouped by connections and/or users. It does not contain information about all memory, only on memory, which can be uniquely associated with either a connection, a statement or a user.

To see information about allocated memory by component, use the M_HEAP_MEMORY system view.
