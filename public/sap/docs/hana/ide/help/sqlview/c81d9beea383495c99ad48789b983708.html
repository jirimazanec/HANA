TABLE_PARTITIONS System View

Partition-specific information for partitioned tables.

You need the DATA ADMIN system privilege to see table information, unless you own or have SELECT permission on the tables.

Structure
Column Name Data Type Description
SCHEMA_NAME NVARCHAR(256) Schema name.
TABLE_NAME NVARCHAR(256) Table name.
PART_ID INTEGER Logical partition ID. 1 through number of partitions for partitioned tables. This is the ID shown in all monitoring views.
LEVEL_1_PARTITION INTEGER First level partition ID. 1 through number of first-level partitions. First-level partitions (those used in ALTER TABLE... MOVE) may have subpartitions.
LEVEL_2_PARTITION INTEGER Subpartition ID. 0 for tables without multilevel partitioning and 1 through number of subpartitions for partitioned tables with multilevel partitioning.
LEVEL_1_RANGE_MIN_VALUE NVARCHAR(5000) Minimum value of the range partition at first level for a range-partitioned table; empty otherwise.
LEVEL_1_RANGE_MAX_VALUE NVARCHAR(5000) Maximum value of the range partition at first level for a range-partitioned table; empty otherwise.
LEVEL_2_RANGE_MIN_VALUE NVARCHAR(5000) Minimum value of the range partition at second level for a range-partitioned table; empty otherwise.
LEVEL_2_RANGE_MAX_VALUE NVARCHAR(5000) Maximum value of the range partition at second level for a range-partitioned table; empty otherwise.
IS_CURRENT VARCHAR(5) Indicates the partition is current partition. Valid values are: ''TRUE' or 'FALSE'.
UNIQUE_CONSTRAINTS VARCHAR(5) Indicates if unique constraints will be checked for this partition or not. Valid values are: ''TRUE' or 'FALSE'.
LOAD_UNIT VARCHAR(6) Valid values are: 'PAGE' or 'COLUMN'. Specify 'PAGE' when partition uses paged attributes, 'COLUMN' otherwise.
STORAGE_TYPE VARCHAR(8) 'DEFAULT' or 'EXTENDED'
