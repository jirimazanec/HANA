ALTER REMOTE SOURCE Statement (Access Control)

Modifies the configuration of an external data source connected to SAP HANA database.

Syntax 1
ALTER REMOTE SOURCE <remote_source_name> <adapter_clause> 
   [<credential_clause>]
Syntax 2 - Linked database
ALTER REMOTE SOURCE <remote_source_name> {<refresh_clause> | <drop_clause>}
Syntax Elements
<remote_source_name>

Specifies the identifier of the remote source.

<remote_source_name> ::= <identifier>
<adapter_clause>

Specifies the adapter configuration.

<adapter_clause> ::= 
 [ADAPTER <adapter_name> [CONFIGURATION FILE <configuration_file>] CONFIGURATION <connection_info_string>]
<adapter_name>

Specifies the corresponding adapter, and the type of access method, to be used by the SAP HANA database to access the data.

<adapter_name> ::= <identifier>
<configuration_file>
Specifies the configuration file for the specified adapter.
<connection_info_string>

Specifies the connection parameters for a given adapter.

<connection_info_string> ::= 
    'webhdfs.url=http://<host_name>:50070/;webhcat.url=http://<host_name>:50111' 
    | 'Driver=<ODBC_library>;ServerNode=<machine_name>:3<instance#>15;'
    | 'DSN=<DSN_entry_name>'
<credential_clause>

Specifies the credential configuration.

<credential_clause> ::= WITH CREDENTIAL TYPE <credential_type> | DROP CREDENTIAL TYPE ['KERBEROS' | 'PASSWORD']
<credential_type>

Specifies a credential type and the credential.

<credential_type> ::= TYPE ['KERBEROS' | 'PASSWORD' [USING <credentials>
<drop_clause>
Delete all linked objects associated with a linked database using the remote source.
<drop_clause> ::= DROP LINKED OBJECTS [CASCASE | RESTRICT]

The CASCADE option drops all the linked tables and dependent objects associated with linked tables. The RESTRICT option drops linked tables are dropped only if there are no dependencies on any of the linked tables. If this option is used and there are dependent objects on a linked table, an error is raised, and all linked tables are retained. If no drop option is specified, all internally generated linked tables with no dependencies are dropped. Linked tables with dependencies are retained.

<refresh_option>
Refresh metadata for a single linked table or all linked objects using the remote source.
<refresh_option> ::= REFRESH LINKED { OBJECTS | TABLE <table_name> }

<table_name> ::= <database_name>.<schema_name>.<identifier>

<database_name> is the name of the remote source. <identifier> is the name of the table on the remote source.

Description

Only database users with the ALTER object privilege for remote sources or the system privilege DATA ADMIN may alter remote sources.

This feature is used with restrictions and/or is extended by the following SAP HANA option:

  • SAP HANA smart data integration

Example

Alter a remote source S by using the ODBC data source ProductionASE that has already been defined in the system. You can also set a new technical user credential for the source.

ALTER REMOTE SOURCE S ADAPTER ASEODBC 
    CONFIGURATION 'DSN=ProductionASE' 
    WITH CREDENTIAL TYPE 'PASSWORD' USING 'user="aseuser";password="asepassword"';

Alter a remote source, S, setting a new technical user credential for the source.

ALTER REMOTE SOURCE S 
    WITH CREDENTIAL TYPE 'PASSWORD' USING 'user="aseuser";password="asepassword"';

Alter a remote source, S, dropping the technical user credential for the source.

ALTER REMOTE SOURCE S DROP CREDENTIAL TYPE 'PASSWORD';

Clear all internally generated objects associated with remote source named myremotesys and drop any dependent objects that reference the linked object.

ALTER REMOTE SOURCE myremotesys DROP LINKED OBJECTS CASCADE

Refresh metadata of all linked objects associated with remote source myremotesys.

ALTER REMOTE SOURCE myremotesys REFRESH LINKED OBJECTS DEFINITION

Refresh metadata for remote table myschema.mytable on remote source myremotesys.

ALTER REMOTE SOURCE myremotesys REFRESH LINKED TABLE DEFINITION myremotesys.myschema.mytable
